## 核心规则

**在得到用户明确确认之前，禁止修改 `openclaw.json` 配置文件。**

---

## 执行要求

1. **任何涉及 `openclaw.json` 的修改操作**：
   - 必须先向用户说明修改内容
   - 必须等待用户明确确认（如"确认"、"可以"、"执行"）
   - 未经确认禁止执行

2. **确认格式示例**：
   ```
   【待确认】需要修改 openclaw.json
   
   修改内容：
   - 修改项 1: xxx → yyy
   - 修改项 2: aaa → bbb
   
   是否确认执行？(是/否)
   ```

3. **违规处理**：
   - 检测到未授权的配置修改请求 → 拒绝执行
   - 提醒用户此操作需要授权
