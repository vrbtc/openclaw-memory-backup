# 阿里云百炼用量查询流程

## 📋 任务说明

**触发条件**：用户说"查阿里云套餐用量"或类似指令

**目标**：查询阿里云百炼控制台用量消耗情况并汇报

---

## 🔧 操作流程（优化版 v2）

### 步骤 1：打开浏览器（关键优化）
```
browser action=open
target=host  ← 必须用 host，不要用 profile=openclaw
url=https://bailian.console.aliyun.com/cn-beijing/?tab=coding-plan#/efm/detail
```

**⚡ 性能要求**：整个查询流程 ≤5 秒

**❌ 禁止行为**：
- 不要用 `profile=openclaw`（容易 tab not found）
- 不要先打开首页再导航（多一步）
- 不要截图确认（screenshot 比 snapshot 慢）

**✅ 正确做法**：
- 直接用 `target=host` 打开浏览器
- 直接导航到用量页面 URL
- 直接用 `snapshot` 获取数据（不要用 screenshot）

### 步骤 2：获取页面数据（用 snapshot）
```
browser action=snapshot
targetId=<从上一步获取>
refs=aria
```

**❌ 禁止**：不要用 screenshot（慢且无法提取结构化数据）

### 步骤 3：解析用量数据
从 snapshot 结果中提取：
- 近 5 小时用量百分比
- 近一周用量百分比
- 近一月用量百分比
- 各时间范围的重置时间
- 最后统计时间

### 步骤 4：错误处理
| 错误 | 处理方式 |
|------|----------|
| tab not found | 重试时用 target=host |
| 页面加载超时 | 最多重试 2 次，每次 5 秒 |
| 未登录 | 走登录流程（见下方） |
| snapshot 为空 | 等待 2 秒后重试 |

2. 输入账号：`13186157868`
3. 输入密码：`Aa2599589`

**情况 C：出现验证码**

## ⚠️ 注意事项

1. **验证码处理**：出现验证码时自动点击发送，然后问用户要
2. **简化汇报**：只汇报用量消耗表格，其他信息不汇报
3. **登录状态**：先检查是否已登录，避免重复登录
4. **浏览器 target**：必须用 `target=host`，禁止用 `profile=openclaw`
5. **数据获取**：用 `snapshot` 不要用 `screenshot`（快 3 倍）
6. **性能要求**：整个流程 ≤5 秒，超时立即重试

## 📊 性能复盘（2026-03-12）

**问题**：用户反馈查询耗时过长

**根因**：
1. 用 `profile=openclaw` 导致 tab not found（浪费 2 次请求）
2. 多次截图确认页面状态（screenshot 比 snapshot 慢）
3. 先打开首页再导航到用量页（多一步）

**优化后**：
- 直接用 `target=host` → 避免 tab not found
- 直接用 `snapshot` → 无需截图，直接获取结构化数据
- 直接导航到用量页 → 减少 1 步操作

**预期性能**：5 秒内完成查询
