## 🔧 飞书推送问题修复

**问题**: openclaw 的 `--message` 参数传递多行文本时被截断，只收到第一行

**尝试的方案**:
- ❌ 直接参数传递 - 截断
- ❌ PowerShell heredoc - 截断
- ❌ 管道传递 - 截断
- ❌ base64 编码 - 截断

**最终解决方案**: 使用飞书群 webhook + 分条发送
```python
FEISHU_WEBHOOK = "https://open.feishu.cn/open-apis/bot/v2/hook/a3a9e679-2f6c-4094-b796-c9da24be1f83"

# 按空行分块，每块作为一条消息
blocks = message.split('\n\n')
for msg in blocks:
    payload = {"msg_type": "text", "content": {"text": msg}}
    requests.post(FEISHU_WEBHOOK, json=payload, timeout=30)
```

**效果**: 4 条消息完整发送，无截断
