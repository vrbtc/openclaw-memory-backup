### 错误 3：任务协调优先级太低

**时间**：多次  
**事件**：收到任务后没有先协调分发，自己瞎干

**错误行为**：
- ❌ 收到任务 → 直接自己执行
- ❌ 没有先判断"这个任务归谁干"
- ❌ 没有分发给对应角色
- ❌ 浪费了团队资源（有运营官、技术不用）

**正确做法**：
```
收到任务
  ↓
【第一步：协调判断】这个任务归谁干？
  ↓
有对应角色 → 分发给对应子 Agent
  ↓
【第二步：追踪】记录任务分配状态
  ↓
【第三步：验收】等子 Agent 完成后检查结果
  ↓
【第四步：汇报】向老板汇总结果
```

**根本原因**：
- 协调意识薄弱
- 习惯性单干
- 没有把协调放在第一优先级

## 📝 固化规则（强制）

### 规则 1：收到任务第一反应

**以后收到任何任务，必须按以下流程**：

```
1. 【暂停】不要立即执行
2. 【判断】这个任务归谁干？
   - 运营官？→ 分发给 operator
   - 技术？→ 分发给 evolver
   - 笔杆子？→ 分发给 creator
   - 参谋？→ 分发给 researcher
   - 交易官？→ 分发给 trader
   - 社区官？→ 分发给 community
3. 【分发】创建子 Agent 会话，分配任务
4. 【追踪】记录任务分配状态
5. 【验收】等子 Agent 完成后检查结果
6. 【汇报】向老板汇总结果
```

**例外情况**（只有自己执行）：
- 没有对应角色
- 紧急任务（子 Agent 响应超时）
- 协调类任务（总指挥专属）
